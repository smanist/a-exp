# Task Lifecycle

Tasks begin open and end complete. Do not mark partial work as complete.

Use:

- `[blocked-by: ...]` for external blockers outside agent control
- `[zero-resource]` for work that should not consume budgeted resources

Prefer mid-sized coherent tasks with mechanical `Done when` criteria. A task may have a single `Done when` line or a multi-bullet checklist.

Split a task only when the parts are independently useful, independently verifiable, need separate approval or resource gates, or are unlikely to fit in one agent session. Merge related steps when they are part of one cohesive outcome and share the same verification surface.
